git diff로 현재 변경사항을 확인하고 다음 관점에서 코드 리뷰를 수행해주세요:

## 리뷰 항목

1. **코드 품질**: 가독성, 명명 규칙, 코드 구조
2. **잠재적 버그**: 엣지 케이스, 널 체크, 타입 안전성
3. **성능 이슈**: 불필요한 연산, 메모리 누수 가능성
4. **보안 취약점**: 인젝션, XSS, 민감 정보 노출

## 출력 형식

각 항목에 대해:
- 심각도: 높음/중간/낮음
- 위치: 파일명:라인번호
- 설명: 문제점과 개선 방안
