# PR 작성 워크플로우

PR 작성을 목표로 계획을 수립하고 실행합니다.

## Phase 1: 계획 수립

### 1. 변경 범위 정의
- 이 PR의 목적은 무엇인가요?
- 어떤 문제를 해결하나요?
- 어떤 기능을 추가하나요?

### 2. 영향 분석
다음을 탐색하세요:
- 수정할 파일 목록
- 영향받는 기존 기능
- 필요한 테스트

### 3. 구현 계획
```markdown
## PR 계획

### 제목
[conventional commit 형식]

### 요약
[1-3 bullet points]

### 변경 파일
1. `path/to/file.ts` - 변경 내용
2. ...

### 테스트 계획
- [ ] 테스트 항목 1
- [ ] 테스트 항목 2
```

### 4. 계획 확인
"이 계획으로 PR을 작성할까요?"

---

## Phase 2: 실행

계획 승인 후:

1. **Auto-accept 모드 권장**: `Shift+Tab`으로 전환
2. 계획대로 순차 구현
3. 각 단계 완료 시 체크
4. 모든 변경 완료 후 PR 생성

---

**Tip**: 좋은 계획 = 1-shot 완성 가능
